Github账户在SSH下的切换

以下命令均在git-bash下执行，如果是cmd下，需将~/替换成C:/Users/username/

  • 首先移除已配置的Github账户：

    git config --global --unset-all user.name
    git config --global --unset-all user.email

  • 进到ssh目录，生成两个账户的密钥等：

    cd ~/.ssh
    ssh-keygen -t rsa -C "email1@example.com" -f name1
    ssh-keygen -t rsa -C "email2@example.com" -f name2

  • 生成结果如下图所示，然后分别将pub后缀的公钥粘贴到github中。

  • 此后将两个都用ssh-add name(1或2)加入，再按照后面内容，我并不能实现账户的切换。因此我实际操作是：

    • 用ssh-add -D删除已有的keys
    • 再将需要的用的账户用ssh-add name加进去
    • 然后使用ssh -T git@github.com验证是否连接到github账户，同时如果你的.ssh目录下没有known_hosts文件，它会提示你，并让你选择是否生成。

  • cmd下可能会报错，需要自己输入以下命令生成known_hosts文件，使得ssh能够成功连接github。

    ssh-keyscan -t rsa github.com >> C:\Users\username\.ssh\known_hosts

  • 第一次使用ssh-add时，可能会遇到connect error类似的错误，这是因为ssh-agent没有打开。只要将计算机管理下的服务中的OpenSSH服务，设置为自动。然后执行ssh-agent bash

总结

到此为止，我们能够成功地随意切换ssh连接的github账户。唯一不足的便是每次都需要删除keys，并将需要使用的账户的key添加进去。

网上看了很多博客，说到config配置，但是具体怎么切换账户并没有说明，尝试了也没有成功。
